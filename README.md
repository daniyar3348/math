# BilimHub

Двуязычная (қазақша / русский) образовательная платформа: публичный каталог
курсов и челленджей, диагностические тесты, LMS (курсы → модули → уроки →
задания), банк вопросов 10 типов с конструктором тестов, геймификация
(баллы, уровни, серии, бейджи, рейтинги) и полная административная панель —
администратор создаёт и публикует всё через UI, без изменения кода.

Название, логотип, цвета и контакты меняются в «Админка → Настройки»
(BilimHub — рабочее название по умолчанию).

## Требования

- Node.js ≥ 20 (проверено на 25)
- pnpm ≥ 9 (проверено на 11)
- Docker (для PostgreSQL 16)

## Быстрый старт

```bash
cp .env.example .env          # секреты/подключения (dev-значения уже валидны)
docker compose up -d          # PostgreSQL 16 на localhost:5433
pnpm install
pnpm db:migrate               # миграции Prisma (dev) — или pnpm db:deploy
pnpm db:seed                  # роли, аккаунты, демо-контент (kk+ru)
pnpm dev                      # http://localhost:3000
```

Проект запускается одной цепочкой команд выше; продакшен-сборка — `pnpm build && pnpm start`.

## Тестовые аккаунты (только для разработки)

| Роль | Логин | Пароль |
|---|---|---|
| Администратор (+SUPER_ADMIN) | `admin@bilimhub.local` | `Bilim2026!` |
| Преподаватель | `teacher@bilimhub.local` | `Bilim2026!` |
| Родитель | `parent@bilimhub.local` | `Bilim2026!` |
| Ученик | телефон `+77000000001` | одноразовый код (в dev показывается на странице входа) |

Эти пароли предназначены исключительно для локальной разработки.
**Использовать их в production запрещено** — сид отказывается запускаться при
`NODE_ENV=production`, а dev-SMS-провайдер, показывающий OTP на странице,
в production бросает ошибку.

## Переменные окружения

Полный список — в [.env.example](.env.example) (реальных секретов в репозитории нет):

| Переменная | Назначение |
|---|---|
| `DATABASE_URL` | PostgreSQL (по умолчанию `localhost:5433`, БД `bilimhub`) |
| `DATABASE_URL_TEST` | Отдельная PG-схема `test` для интеграционных тестов |
| `APP_SECRET` | HMAC-секрет (подпись файловых URL, webhook подписи) |
| `SMS_PROVIDER` | `dev` (код виден локально) или `http` (+ `SMS_HTTP_URL`, `SMS_HTTP_TOKEN`) |
| `PAYMENT_PROVIDER` | `mock` — реальные ключи не хранятся в репозитории |
| `FILE_STORAGE`, `UPLOADS_DIR` | Диск (интерфейс готов к S3) |
| `TZ` | `Asia/Almaty` — часовой пояс платформы |

## Команды

```bash
pnpm dev          # dev-сервер
pnpm build        # production-сборка
pnpm start        # запуск production-сборки
pnpm lint         # ESLint
pnpm typecheck    # tsc --noEmit
pnpm test         # vitest: unit + integration (отдельная PG-схема)
pnpm test:e2e     # Playwright E2E (18 сценариев, включая axe-a11y)
pnpm db:migrate   # prisma migrate dev
pnpm db:deploy    # prisma migrate deploy (production)
pnpm db:seed      # сид: роли, аккаунты, демо-контент
```

Перед первым `pnpm test:e2e` выполните `pnpm exec playwright install chromium`.

## Запуск через Docker Compose (полностью)

```bash
cp .env.docker.example .env.docker   # задать APP_SECRET и пароль БД
docker compose --profile app up -d --build
```

Приложение примигрирует БД при старте и поднимется на `:3000`
(сид в production-режиме не выполняется — аккаунты создаются вручную).

## Что реализовано

- **Публичная часть**: лендинг (hero, направления, преимущества, форматы,
  популярные курсы, активные челленджи, «как работает», отзывы, FAQ, контакты,
  политика/соглашение, CTA); каталоги курсов и челленджей с фильтрами
  (статус/цена/предмет/класс/поиск) в URL, пагинацией, skeleton/empty/error.
- **Языки**: маршруты `/kk/...` и `/ru/...`, переключатель с сохранением
  страницы, таблицы переводов с `locale`, публикация заблокирована без обоих
  языков (индикатор готовности в админке), hreflang/canonical, ₸, Asia/Almaty.
- **Аутентификация**: ученик — телефон+OTP (абстракция `SmsProvider`);
  персонал — email+Argon2id (+опциональный TOTP); ротация refresh-токенов
  с детекцией повторного использования; httpOnly/SameSite cookies; rate limiting
  в БД; журнал входов; «выйти со всех устройств».
- **RBAC на сервере**: SUPER_ADMIN / ADMIN / TEACHER / STUDENT / PARENT,
  каталог разрешений, guard'ы на каждом API; преподаватель ограничен своими
  курсами и не публикует; родитель видит только привязанных детей.
- **Банк вопросов**: 10 типов (один/несколько ответов, верно-неверно, короткий
  текст, числовой с погрешностью, пропуски, сопоставление, порядок, эссе,
  файл), KaTeX-формулы, версионирование, архивирование, дублирование,
  CSV-импорт с построчным отчётом об ошибках и экспорт.
- **Конструктор тестов**: секции, ручной выбор и случайная выборка из банка,
  баллы, проходной %, лимит времени, попытки, окна доступности, перемешивание,
  политика показа результатов/правильных ответов, код доступа, доступ группам,
  платность, режимы диагностика/экзамен, предпросмотр без аналитики.
- **Прохождение**: серверный таймер, автосохранение (+сброс несохранённого
  при завершении), восстановление после перезагрузки, навигация
  отвечен/пропущен/помечен, предупреждение о неотвеченных, идемпотентное
  транзакционное завершение, автооценивание, настраиваемый частичный балл,
  очередь ручной проверки, история ответов, событие смены вкладки.
- **Диагностика**: результат по темам/целям, сильные/слабые стороны,
  рекомендации, понятная диаграмма.
- **LMS**: курсы/модули/уроки (Markdown+медиа), задания с дедлайном и
  пересдачей, проверка преподавателем, группы, запись (админом и самозапись),
  последовательное прохождение, прогресс, объявления, обсуждение, журнал
  оценок с CSV, сертификаты с проверочным кодом (`/[locale]/certificates/[code]`).
- **Челленджи и геймификация**: даты регистрации/проведения, лимит участников,
  платность, код доступа, связанные тесты с весами, призы; баллы с защитой от
  повторного начисления, уровни, streak, бейджи, ручная корректировка только
  с комментарием, скрытие несовершеннолетнего из публичного рейтинга;
  лидерборд пересчитывается фоновой задачей, tie-break документирован
  (баллы ↓ → время финиша ↑ → время записи ↑).
- **Оплата**: интерфейс `PaymentProvider` + `MockPaymentProvider`, статусы,
  идемпотентный webhook с HMAC-подписью, история, ручная выдача доступа.
- **Админка `/admin`**: dashboard с метриками и графиками 30 дней, 17 разделов,
  таблицы с серверной пагинацией/поиском/фильтрами/сортировкой/массовыми
  действиями/CSV-экспортом/URL-состоянием/подтверждениями, workflow
  Черновик → На проверке → Опубликовано → Архив, AuditLog.
- **Безопасность**: zod-валидация (strict), CSP и security-заголовки, Origin-check
  мутаций, санитизация Markdown (экранирование до рендера), подписанные ссылки
  на файлы, лимиты размера/типа файлов, идемпотентность критических операций,
  маскирование секретов в аудите, секреты только в env.

## Тесты

- `tests/unit` — оценивание всех типов (граничные случаи частичного балла),
  TOTP по векторам RFC 6238, безопасный Markdown.
- `tests/integration` — жизненный цикл попытки (resume, идемпотентный finalize,
  ручная проверка с clamp, идемпотентные баллы, лимит попыток), i18n-гейт
  публикации, tie-break лидерборда, платёжный webhook. Выполняются в отдельной
  PG-схеме `test` и не трогают dev-данные.
- `tests/e2e` — 18 Playwright-сценариев §18: полный «золотой путь» (админ через
  UI создаёт вопрос→тест→челлендж и публикует; ученик регистрируется по OTP,
  проходит, получает результат и баллы; админ видит начисление), ручная
  проверка эссе, RBAC (ученик/преподаватель/гость), переключение языка,
  фильтры каталога, axe-smoke WCAG 2.2 AA.
