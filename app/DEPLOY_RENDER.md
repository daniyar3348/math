# Деплой Esep+ на Render (полная версия, ~5 минут)

Render запускает полноценный сервер: реальная БД, аккаунты, админка, анти-чит —
всё как на VPS, но без администрирования. Деплой описан в `render.yaml` (корень репо).

## Один раз: подключение

1. Зайди на **https://render.com** → Sign in with GitHub (аккаунт daniyar3348).
2. **New +** → **Blueprint** → выбери репозиторий **daniyar3348/math**.
3. Render прочитает `render.yaml` и спросит **ADMIN_PASSWORD** — задай сильный пароль
   (это вход админа: email `admin@esep.kz`).
4. **Apply**. Первая сборка ~5–7 минут (docker build).

Готово: сайт на `https://esep.onrender.com` (или `esep-XXXX.onrender.com`,
если имя занято — точный URL покажет дашборд).

Дальше каждый `git push` в `main` автоматически передеплоивает сайт.

## Тарифы — что выбрать

| | Free | Starter ($7/мес) + диск (~$0.25/мес) |
|---|---|---|
| Полный функционал | ✅ | ✅ |
| Засыпание при простое | 😴 15 мин → первый запрос 30–60 с | нет |
| **SQLite-база** | ⚠️ сбрасывается при передеплое/перезапуске | ✅ постоянная |
| Для чего | витрина, показ инвесторам/школам | реальные ученики |

**Переход на постоянную БД:** в `render.yaml` смени `plan: free` → `plan: starter`
и раскомментируй блок `disk:` → пуш → Render применит. Диск монтируется в
`/app/data` — база и бэкапы переживают любые деплои.

## Настройки (уже заданы в render.yaml)

- `ADMIN_EMAIL=admin@esep.kz`, `ADMIN_PASSWORD` — задаёшь при создании
- `SEED_DEMO=1` — демо-ученики в рейтинге (на free БД эфемерна, так живее);
  для прода на Starter поставь `0` в дашборде (Environment)
- `PAYMENTS_MODE=sandbox` — «оплата» демо; продажи вести через ручную выдачу
  курсов в админке, `live` включать только с вебхуком Kaspi
- Health check: `/api/health`; регион: Frankfurt (ближайший к КЗ)

## Свой домен (опционально)

Dashboard → Settings → Custom Domains → добавь `esep.kz` → пропиши CNAME у
регистратора. HTTPS Render выпустит сам.
