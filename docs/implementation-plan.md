# BilimHub — план реализации

Двуязычная (kk/ru) образовательная платформа: LMS (курсы/уроки/задания/оценки)
+ банк вопросов и конструктор тестов + диагностика + челленджи с геймификацией
+ админ-панель, из которой всё создаётся без кода. Бренд (название, логотип,
цвета, контакты) настраивается через SiteSettings.

## Архитектура (итоговая)

```
Next.js 16 (App Router, TS, SSR)  ←  единое full-stack приложение
 ├─ app/[locale]/…        публичные страницы и кабинеты (kk/ru)
 ├─ app/admin/…           админ-панель (RU-first, доступ по ролям)
 ├─ app/api/…             REST API (route handlers) — вся бизнес-логика на сервере
 ├─ lib/                  домены: auth, rbac, i18n, engine (тесты), points, payments…
 └─ prisma/               схема (~45 моделей), миграции, seed
PostgreSQL 16 (Docker)    основная БД
Хранилище файлов          локальный диск через абстракцию FileStorage (S3-ready)
Фоновые задачи            in-process scheduler + идемпотентные пересчёты (интерфейс Queue)
```

Отклонения от рекомендованного стека (NestJS/Turborepo/Redis/MinIO) обоснованы
в `docs/decisions.md` — цель: работающий продукт, а не каркас.

## Этапы

- **P0** Каркас: Next+TS+Tailwind, docker-compose (Postgres), Prisma, env, токены дизайна.
- **P1** Данные и доступ: полная Prisma-схема (§14 ТЗ, таблицы переводов с `locale`,
  multi-tenant `organizationId`, soft delete, аудит), миграции, базовый seed
  (организация, роли, 4+1 аккаунтов), auth (Argon2id, сессии+ротация refresh,
  OTP для учеников через `SmsProvider`, опц. TOTP для админа), RBAC `can()` на сервере,
  i18n-инфраструктура `/kk|/ru`, rate limiting, AuditLog.
- **P2** Движок: банк вопросов (10 типов, версии, статусы, импорт/экспорт CSV),
  автооценивание (+частичный балл), конструктор тестов (секции, случайные выборки,
  все настройки), попытки (серверный таймер, автосохранение, resume, идемпотентное
  завершение, навигация/флаги), ручная проверка, диагностический отчёт по темам,
  баллы/уровни/серии/бейджи, лидерборды с tie-break, челленджи, LMS
  (курсы→модули→уроки→задания→проверка→прогресс→журнал→CSV→сертификаты),
  Mock-платежи с идемпотентным webhook, уведомления, файлы с подписанными URL.
- **P3** UI: дизайн-токены (фиолетовый/жёлтый, настраиваемые), UI-kit + все состояния,
  лендинг (управляется из админки), каталоги с фильтрами в URL, страницы челленджей
  по slug, кабинеты ученика/преподавателя/родителя, экран прохождения теста,
  результаты и диаграммы, a11y (skip-link, фокус, ARIA, reduced-motion), SEO+hreflang.
- **P4** Админ-панель: единый табличный фреймворк (серверная пагинация/поиск/фильтры/
  сортировка/URL-state/массовые действия/экспорт/подтверждения), dashboard с метриками,
  все разделы §12, workflow черновик→проверка→опубликовано→архив, индикатор
  готовности перевода, публикация заблокирована без обоих языков.
- **P5** Демо-seed (§19, двуязычный), тесты (vitest: движок оценивания, tie-break,
  идемпотентный finalize, i18n-гейт; Playwright E2E §18 + axe), lint/typecheck/build,
  документация (README, architecture, data-model+ER, rbac, test-engine, i18n,
  deployment, decisions, OpenAPI), финальная проверка в браузере, commit+push.

## Критерий завершения
§21 ТЗ: от «админ создаёт и публикует тест через UI» до «ученик проходит и получает
результат», ручная проверка, RBAC на сервере, оба языка, миграции с чистой БД,
seed-аккаунты, отсутствие критических ошибок TS/lint/тестов.
