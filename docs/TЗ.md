## 1. Цель и суть продукта

Веб-платформа для подготовки школьников к **поступлению в БИЛ, НИШ и КТЛ** через
**математику**. На старте — **только математика** (профильный вступительный предмет
этих школ). Ключевая идея — обучение через **геймификацию**: челленджи, XP-очки, медали,
рейтинги (лидерборды), интерактивные уроки и тесты. Доступ — **freemium**: часть контента
бесплатно, продвинутые курсы/тренажёры — платно (разовые покупки).

Аналог: zerdeli.app — раздел `student/challenges`.

### 1.1. Целевые школы (треки подготовки)
- **БИЛ** — Білім-инновация лицейлері (казахско-турецкие лицеи, физ-мат уклон).
- **НИШ** — Назарбаев Интеллектуальные Школы.
- **КТЛ** — Қазақ-Түрік лицейлері.

Каждая школа = отдельный «трек» с курсами и челленджами под формат её вступительного
экзамена по математике (логика, арифметика, алгебра, геометрия, нестандартные задачи).

## 2. Целевая аудитория

- Школьники ~5–7 классов, готовящиеся к поступлению в БИЛ / НИШ / КТЛ.
- Их родители (покупают доступ).
- Предмет на старте — **только математика**. Другие предметы — будущие этапы.

## 3. Язык интерфейса

## 5. Основные модули (по аналогии с Zerdeli)

### 5.1. Челленджи (`/student/challenges`) — ядро продукта
- Список челленджей по предметам и курсам.
- Каждый челлендж = набор вопросов/заданий с таймером.
- Начисление **XP** за прохождение, медали/достижения.
- Статистика прохождения (сколько решено, % правильных, заработанный XP).
- Типы вопросов: одиночный выбор, множественный выбор, ввод ответа. `[?]` Добавить аудио/изображения.

### 5.2. Тренажёры (Practice)
- Режим бесконечной практики по темам без таймера.
- Разбор правильных ответов.

### 5.3. Уроки / Курсы
- Интерактивные уроки (текст, изображения, аудио, видео `[?]`).
- Объединены в курсы по предметам (Математика, Физика, История и т.д.).
- Часть бесплатно, часть — платно.

### 5.4. Геймификация
- XP-очки за активность.
- Медали/бейджи за достижения.
- Ежедневные задания (daily challenges) `[?]`.
- Streak (серии дней) `[?]`.

### 5.5. Лидерборд (рейтинг)
- Рейтинг студентов по XP.
- Фильтры: по школе / району / региону / всему Казахстану (как у Zerdeli).
- `[?]` Период: за всё время / месяц / неделя.

### 5.6. Личный кабинет студента
- Профиль (имя, школа, регион, класс).
- Прогресс, заработанный XP, медали.
- Купленные курсы, история активности.

## 6. Система оплаты (ключевое требование)

### 6.1. Модель монетизации — ✅ РЕШЕНО
- **Разовая покупка отдельных курсов** (как Zerdeli, ~3 990 ₸/курс).
- Часть курсов бесплатна (freemium-витрина), часть платная.
- Подписку можно добавить позже как отдельный этап.

### 6.2. Платёжный провайдер — ✅ РЕШЕНО: Kaspi Pay
- Требуется: договор с Kaspi и эквайринг (Kaspi Business / Kaspi Pay API).
- Для разработки до получения боевых ключей — **mock/sandbox-режим оплаты**
  (кнопка «Оплатить» эмулирует успешный платёж и выдаёт доступ), затем подключаем
  реальный Kaspi Pay по их API + вебхуки/redirect-подтверждение.
- `[?]` Уточнить у Kaspi: REST API оплаты, формат вебхука, фискализация чека.

### 6.3. Функционал оплаты
- Страница checkout (выбор курса → оплата картой).
- Вебхуки от провайдера → активация доступа к курсу.
- История платежей в кабинете и в админке.
- Чеки/фискализация `[?]` (по требованиям РК — ОФД/чеки).
- Промокоды/скидки `[?]`.

## 7. Технологический стек — ✅ РЕШЕНО: Next.js + Supabase

- **Framework:** Next.js (App Router) + TypeScript + TailwindCSS.
- **БД + Auth + Storage:** Supabase (PostgreSQL, Auth, файловое хранилище для медиа).
- **Авторизация:** Supabase Auth (email/пароль + телефон/OTP `[?]`).
- **Доступ к данным:** Supabase client + Row Level Security (RLS) для разграничения ролей.
- **Локализация:** next-intl (или аналог) для KK/RU.
- **Платежи:** Kaspi Pay (sandbox-mock → реальный API), серверные route handlers + вебхуки.
- **Хостинг:** Vercel (frontend) + Supabase (managed). `[?]` Домен .kz.
