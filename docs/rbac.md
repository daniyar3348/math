# RBAC

Роли и каталог разрешений определены в [lib/rbac.ts](../lib/rbac.ts); проверка —
только на сервере (`requirePermission` в каждом API-маршруте, guard-функции в
доменном слое). Скрытие кнопок в UI — косметика, не защита.

## Роли

| Роль | Суть |
|---|---|
| SUPER_ADMIN | всё, что ADMIN, + создание администраторов, глобальные настройки |
| ADMIN | пользователи/группы, весь контент и его публикация, баллы, платежи, лендинг, аудит |
| TEACHER | черновики уроков в назначенных курсах, проверка работ, успеваемость своих учеников; **не публикует** и не видит общий список пользователей |
| STUDENT | обучение, тесты, челленджи, свои результаты |
| PARENT | просмотр привязанных детей; **не видит правильные ответы активных тестов** |

## Точки контроля за пределами матрицы

- `assertCourseScope` — преподаватель работает только с курсами, где есть
  запись `CourseTeacher` (IDOR-защита в редакторе уроков и проверке работ).
- `assertParentScope` — родитель читает только детей из `StudentParent`.
- `adminUserAction`: нельзя снять роль с самого себя (`cannot_demote_self`) и
  заблокировать себя; назначить роль ADMIN может только SUPER_ADMIN;
  блокировка и сброс пароля отзывают все сессии пользователя.
- Статусы контента меняются через `/api/admin/status` с проверкой
  `*.publish`-разрешения и i18n-гейта (см. [i18n.md](i18n.md)).
- Родительский дашборд не включает правильные ответы: политика показа
  (`showCorrect`) применяется на сервере при сборке результата.
